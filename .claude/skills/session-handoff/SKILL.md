# session-handoff — проверка переноса сессии (D-0050)

Цель: следующая сессия обязана восстановиться из файлов, без этого
чата. Проверка симметрична загрузке: всё, что Boot прочитает утром,
handoff проверяет вечером.

## Шаги

1. **Git, оба репо** (это репо и D:\AO3_tests, если сессия его
   трогала): `git status --short` пуст — незакоммиченное коммитится
   сейчас; `git log origin/main..main --oneline` (AO3: `master`)
   пуст — иначе `git push`. Незнакомые untracked-файлы не заметать в
   коммит молча — разобраться.
2. **Журнал закрыт** (logs/routing-log.jsonl): нет `delegated` без
   парного `accepted`/`rejected`; нет `lead_degraded` без
   `lead_restored` — деградация, переживающая сессию, обязана быть
   последним событием журнала (D-0039). Каждый ОТКРЫТЫЙ delegated
   сверяется с реальностью (D-0076/F-44): воркер по worker_ref жив
   или результат ждёт приёмки — иначе это фантом (запись без
   запуска); фантом закрывается пометкой в notes следующего события
   и, если работа нужна, честным ре-диспатчем. События сессии
   написаны Edit-тулом (командная гигиена, п.5).
   2а. **Персистентность принятых деливераблов (F-50)**: для каждого
   accepted(builder) СЕССИИ несущая сущность witness (тест/функция/
   файл из witness-поля) существует в закоммиченном HEAD
   (`git show HEAD:<путь>` / grep по HEAD), не только в рабочем
   дереве — широкий batch-коммит или checkout мог смыть
   незакоммиченный дифф между приёмкой и коммитом (класс
   «деливерабл-дрейф», прецедент AO3 AT-BUG-022). Сверка ДО push;
   расхождение чинится тем же вечером: ре-коммит из рабочего дерева,
   а при утере — честный `defect_found` (ref на исходный accepted),
   не тихий перезапуск. Недельная сеть — чек 31 калибровки.
3. **CURRENT_CONTEXT.md**: закрытые работы заархивированы в
   docs/task_reports/ (D-0038), очередь актуальна. Тест переноса:
   «всё, что знаю только я, записано в файлы?» — правило
   SESSION_PROTOCOL: важное знание не остаётся только в чате.
   3а. **Кросс-деплойные пункты (D-0082)**: каждое обязательство,
   созданное сессией для ДРУГОГО деплоя, записано в носитель ЦЕЛИ
   (AO3: docs/HANDOFF.md; OS: CURRENT_CONTEXT.md) — notes своего
   журнала/FINDINGS не считаются передачей; непереданный пункт —
   явная строка «не передан» в своём носителе, FAIL до закрытия.
4. **Boot-бюджет**: `wc -l -c` по CLAUDE.md + списку файлов BOOT.md.
   Сравнить с прошлым замером (notes последнего события `calibrated`
   или прошлого handoff-отчёта). Флаги: рост >10% за сессию без
   объяснимой причины; суммарный размер >100 КБ. Пробой → прогнать
   скилл boot-diet ТЕМ ЖЕ вечером (D-0068; он — единственный
   владелец порядка реакции: сначала архивная развёртка и проверка
   пар урезанное↔полное, глубокие срезы — только решением
   Lead+Architect).
5. **Живость boot-цепочки**: каждый файл из списка BOOT.md существует
   (переименование без правки BOOT.md = сломанный boot следующей
   сессии).
6. **Итог**: короткий отчёт по пунктам 1–5 (OK/FAIL и что сделано).
   FAIL чинится ДО закрытия сессии, а не записывается «на потом».
   Финальный коммит и push — последнее действие сессии.

Детектор пропуска этого чека — Boot Report СЛЕДУЮЩЕЙ сессии
(BOOT_REPORT_PROTOCOL.md, правило 6): грязное дерево или расхождение
с origin на старте = прошлая сессия закрылась без handoff-проверки;
это фиксируется как находка, а не молча поглощается.
