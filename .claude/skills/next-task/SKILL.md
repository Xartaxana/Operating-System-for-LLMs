# /next-task — первая по приоритету задача

Команда задаёт ПОРЯДОК шагов. Правила — только в CLAUDE.md
(автозагружен); при расхождении CLAUDE.md выигрывает, сюда правила
не дублируются.

0. **Ярус (D-0056а, D-0058).** Сверь фактическую модель с Fable.
   Ниже → работай по матрице «Роль ≠ ярус»: координируй и диспатчь;
   принимай только ярусы ниже своего (равный — только через
   critic-вход); Lead-класс (механизмы, DECISIONS, статусы, гейты,
   приёмки равного/высшего яруса) — копи явными строками в очередь
   полного Lead в CURRENT_CONTEXT.md; окно деградации журналируй,
   если применимо (D-0039). Команда исполнима с любого яруса — она
   не делает сессию Lead'ом.
1. **Контекст.** Если boot этой сессии не выполнялся — сначала
   BOOT.md + Boot Report; вызов этой команды считается явным
   подтверждением оператора (правило 4 протокола Boot Report
   удовлетворено), отдельного «да» не жди.
2. **Приоритет.** (а) Current Task в CURRENT_CONTEXT.md назначена
   (D-0025) → она первая. (б) Иначе — первый ИСПОЛНИМЫЙ пункт
   Remaining Lead-tier Queue с учётом записанных там рекомендаций.
   (в) Пункты, требующие решения оператора (подпись гейта, выбор
   направления), не бери — отметь в отчёте и возьми следующий.
   Выбор назови в первом сообщении хода.
3. **Диспатч вниз — сразу и в фон (D-0040).** Части задачи,
   отображающиеся на ярусы воркеров, отдай ДО начала собственной
   части: каждому — спека с DoD (D-0054); task_id — перечитыванием
   хвоста журнала (D-0060); параллельным — владение путями;
   событие `delegated` до запуска.
4. **Пока воркеры работают** — делай координаторскую часть сам;
   между приёмками можно закрывать мелкие точечные пункты очереди
   (само-чтение — по правилу 1, с dispatch_skipped).
5. **Приёмка по завершении каждого воркера** — по форме яруса
   (правила 1–3: след/witness/critic) и по матрице шага 0.
   Отклонение → `rejected` с failure_class; две неудачи на ярусе →
   эскалация (правило 6).
6. **Коммиты по ходу (D-0016):** одна концептуальная правка — один
   коммит; механизмные пути — правило 10 + осевой блок (D-0055);
   правка role-файла яруса с golden set — прогон до коммита
   (D-0057).
7. **Финал хода:** краткий отчёт (сделано / в очередь / ждёт
   оператора), CURRENT_CONTEXT обновлён. Новую задачу сверх
   выбранной не начинай (D-0025); исключение — пункты, явно
   помеченные в очереди «на касании», если касание случилось.

Детектор отказа (правило 10в): команда лишь упорядочивает
существующие механизмы — её выход (события журнала, коммиты,
CURRENT_CONTEXT) аудируется чеками 1/3/6/13 калибровки; выбор
задачи против приоритета виден следующей сессии как расхождение
Current Task ↔ фактические коммиты (Boot Report, D-0025).
