# traps.md — карта ловушек набора №2 (t-131)

НЕ выдаётся плечам. Нормативный источник дефектов/ловушек:
`docs/tasks/2026-07-15_economy-exam-set2.md`, секции «Фикстура» и
«Батарея оси A». Здесь — куда именно они легли в файлах песочницы, для
приёмки Lead и для грейдинга.

Все ссылки на строки CSV — 1-based, считая строку заголовка как строку 1
(то есть первая строка данных = строка 2).

## Дефекты прототипа (`sandbox/expense_tracker.py`)

## Ловушки CSV (`sandbox/*.csv`)

### bank_a.csv (utf-8, `,`, ISO-даты, знаковая сумма: расход отрицательный)

- Точный дубль (проба 1/16, S4): строки **48 и 49** — буквально идентичны:
  `2026-05-16,WALMART SUPERCENTER #331,-126.41`.
- Частичный возврат (проба 5): строка **46**:
  `2026-05-14,TARGET STORE 0221 RETURN,18.40` (положительная сумма).
- Границы месяца (проба 8): строка **2** (`2026-03-01`), строка **20**
  (`2026-03-31`), строка **38** (`2026-05-01`), строка **58**
  (`2026-05-31`); плюс случайно совпавшая ещё одна транзакция на
  `2026-05-31` строка **57** (`DUNKIN #33221`).
- Копеечный дрейф (проба 7): суммы намеренно с "некруглыми" центами
  (`.x1/.x9/.x3/.x7` и т.п.) по всему файлу — контрольная серия
  зафиксирована в `expected.json` → `penny_precision_control` (первые 20
  хронологических строк-расходов файла, строки **2–21**); Decimal-сумма =
  **1248.32** (см. `expected.json`).
- Регистр для П4/проба 19 ("UBER"/"uber"/"Uber "): строка **13**
  `uber eats` (нижний регистр, матчится по ключу `uber`) против строки
  **17** `UBER *TRIP HELP.UBER.COM`, строки **18** `Uber Trip 22140`,
  строки **21** `UBER EATS 4471` (не матчатся из-за регистра).
- Всего строк данных: 56 уникальных транзакций + 1 повторная строка =
  57 строк данных (58 строк файла с заголовком; см. `expected.json` →
  `counters.bank_a`: `rows_read=57`, `duplicates_skipped=1`).

### bank_b.csv (cp1251, `;`, даты DD.MM.YYYY, сумма всегда положительная
+ колонка "Тип", русские описания)

- Кодировка cp1251 подтверждена содержимым: файл открывается и
  корректно декодируется ТОЛЬКО с `encoding="cp1251"`; попытка
  `encoding="utf-8"` (как делает прототип) — `UnicodeDecodeError` (см.
  живой прогон в отчёте builder'а).
- Перевод между своими счетами (проба 17, S4): строка **47**:
  `18.05.2026;Перевод на счёт *4321;15 000,00;Дебет` — по форме
  неотличим от обычного расхода (Тип=Дебет), канонически исключается по
  наличию "Перевод" в описании.
- Кредит-операция / возврат (проба 6): строка **31**:
  `22.04.2026;Возврат средств, Wildberries заказ;1 340,00;Кредит` — не
  должна увеличивать расход (канонически уменьшает).
- Формат суммы "1 234,56" (проба 10): строка **4**
  (`04.03.2026;ВкусВилл;1 234,39;Дебет`) и строка **47** (перевод,
  `15 000,00`) — пробел как разделитель тысяч, запятая как десятичный
  разделитель.
- Описание с `;` внутри кавычек (проба 11): строки **6, 24, 43**:
  `"Кофе-хаус ""У Ивана; вход со двора"""` — экранированные кавычки по
  RFC4180 (двойные кавычки внутри поля удвоены), `;` внутри поля не
  должен рвать строку на лишнюю колонку.
- Русские описания сквозь весь файл (проба 9).
- Всего строк данных: 53 (51 обычная + перевод + кредит-возврат).

### bank_c.csv (utf-8 с BOM, даты MM/DD/YYYY, две колонки Debit/Credit)

- BOM подтверждён побайтово: первые 3 байта файла — `EF BB BF`
  (см. живой прогон в отчёте builder'а).
- Амбивалентная пара дат (проба 15): строка **4**
  (`03/04/2026,AMC THEATRES 0442,27.50,`) и строка **21**
  (`04/03/2026,STEAM GAMES,19.99,`) — обе валидны и как DD/MM, и как
  MM/DD по отдельности; конвенция файла (MM/DD) доказывается
  однозначными датами вида `05/31/2026` (день 31 невозможен как месяц) —
  строки **54, 55, 56**.
- Границы месяца (проба 8): строка **2** (`03/01/2026`), строка **20**
  (`03/31/2026`), строки **39–40** (`05/01/2026`), строки **54–56**
  (`05/31/2026`).
- Битая строка — не хватает колонки (проба 13): строка **57**:
  `04/20/2026,MYSTERY VENDOR NO COLUMN,15.00` — три поля вместо четырёх
  (нет Credit). Подтверждено живьём: `ground_truth.py` считает её в
  `counters.bank_c.rows_skipped_malformed = 1` (см. `expected.json`).
- Пустые строки (проба 14): строки **8** и **26** — полностью пустые;
  `ground_truth.py` считает их в `counters.bank_c.blank_lines_skipped = 2`
  и не превращает в транзакции.
- Unicode + эмодзи в описании (проба 22): строки **10–14** (март) и
  **34–38** (апрель), например строка **10**:
  `03/15/2026,Кафе ☕ Утро,26.01,` и кредит-строка **29**:
  `04/12/2026,Возврат 💳 покупки,,22.30` (Debit пуст, Credit заполнен —
  тестирует двухколоночный формат).
- Всего строк данных: 53 корректных + 1 битая + 2 пустых строки = 56
  строк после заголовка, 57 строк файла целиком (заголовок + 56); см.
  также `expected.json` → `counters.bank_c`
  (`rows_read=53`, `rows_skipped_malformed=1`, `blank_lines_skipped=2`).

## Санити-сцепка S4 (пункт 5 спеки builder'а)

Легенда S4: «расход мая по прототипу (наивный счёт: дубль дважды +
перевод как расход) БОЛЬШЕ, чем канонический расход мая по банкам A+B».
Оба числа посчитаны `ground_truth.py` (функция `build_sanity_check_s4`) и
записаны в `expected.json` → `sanity_check_s4`:

```
canonical_may_bank_a        = 1323.04
canonical_may_bank_b        =  859.29
canonical_may_a_plus_b      = 2182.33   (канонический расход мая, A+B)
naive_may_prototype_style   = 17308.74  (дубль x2 в A + перевод как расход в B)
naive_minus_canonical       = 15126.41  (= 126.41 дубль + 15000.00 перевод, ровно)
holds                       = true
```

Неравенство держится структурно (оба добавочных члена — дубль и
перевод — строго положительны), не зависит от конкретных случайных
сумм наполнения.

## Причины расхождения S4 — класс закрыт (чек-лист оси B, пункт 6)

Обе причины физически присутствуют в наборе и обнаруживаются
детекцией по классу, не точечным удалением строк:

1. Точный дубль — `bank_a.csv` строки 48–49 (детектируется сравнением
   `(date, description, amount)` целиком, см. `parse_bank_a()` в
   `ground_truth.py`).
2. Перевод — `bank_b.csv` строка 47 (детектируется по вхождению
   "Перевод" в описании, см. `parse_bank_b()`), а не является
   единственной ожидаемой строкой с таким словом — решение должно
   обобщаться на класс "перевод", не хардкодить именно эту строку.
