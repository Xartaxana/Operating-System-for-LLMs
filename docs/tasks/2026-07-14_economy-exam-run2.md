«деплой v0.2.0 + дизайнер» (слово оператора); финальная трактовка

## Правила прогона

1. МОДЕЛЬ: Fable во всех девяти сессиях (сопоставимость с №1).
2. Одна сессия = один каталог = одна задача. Других работ в песочницах
   не вести. Сессию не продолжать после вердикта.
3. Порядок плеч — НОВАЯ ротация (не совпадает с №1; №1 был
   T1 A-C-B / T2 B-A-C / T3 C-B-A):
   - **T1 — B, A, C**
   - **T2 — C, B, A**
   - **T3 — A, C, B**
   (каждое плечо по разу в каждой позиции; порядок каждой задачи
   отличается от №1 — «нет любимого последнего плеча».)
4. Плечу A — только текст задачи. Плечу B — текст задачи + ДОБАВОЧНАЯ
   СТРОКА (opus-спеки). Плечу C — сначала канонический промпт, затем
   текст задачи, одним сообщением.

### ДОБАВОЧНАЯ СТРОКА к сообщению плеча B (все три задачи, дословно)

> Дополнение к процедуре для этой сессии (слово оператора 2026-07-14):
> точные спеки builder-диспатчей не пиши сам. По каждой нетривиальной
> спеке составь интент-бриф (намерение, DoD уровня результата, границы
> и уже принятые решения) и разверни его в точную спеку диспатчем
> opus-дизайнера (general-purpose, model=opus, лейбл «opus: спека …»)
> с ОБЯЗАТЕЛЬНЫМ блоком развилок. Спеку прими сам (сверка с брифом +
> вердикт по каждой развилке), затем запускай builder по принятой
> спеке. Однострочные правки — без дизайнера.

(Добавочная строка идёт ОТДЕЛЬНЫМ абзацем после текста задачи, тем же

намерения задачи; конфаунд учтён выше.)
