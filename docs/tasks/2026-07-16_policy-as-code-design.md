# Политика-как-код: гейты на передаче хода (дизайн, 2026-07-16)

Заказ оператора (утро 07-16): «повысить соблюдение политики — правила
пишем, их никто не соблюдает; поможет ли "перед передачей хода докажи
соблюдение всех пунктов" и можно ли хук?». Ответ Lead: самоотчёт по
всем пунктам — не поможет (три расхождения самоотчёта с фактами за
сутки: фантомный ping.txt t-132-смока, «validator 0 нарушений» при
no-op, батч-ts задним числом) и дорог (чек-лист на каждый ход =
координационный налог, главный едок по декомпозиции №5/№6).
РАБОТАЮЩАЯ форма (данные №8): хук САМ снимает доказательства в
момент передачи хода — факты, не декларации.

## Принцип

Каждое правило политики, сводимое к детерминированному факту,
компилируется в микро-чек на одной из трёх точек передачи хода.
Несводимое (семантика: 11а вопросы вверх, интерпретации, качество
декомпозиции) остаётся второму эшелону — приёмке ярусом выше,
экзаменам, калибровке (D-0063 по построению). Каждый гейт проверяет
факт, ценный сам по себе, НЕ прокси-декларацию (анти-Goodhart —
урок отвергнутого «маркера готовности»).

## Три точки передачи хода

1. **Ход ВНИЗ — PreToolUse(Task), «диспатч-гейт»**: промпт диспатча
   содержит DoD-блок (маркеры: DoD/критерии приёмки/witness)?
   пишущий/параллельный — манифест (given/owns/non-goals)? лейбл
   description начинается с модели воркера (правило 7)? Нарушение →
   блок с подсказкой. Сдвигает правило 11 из «воркер вернёт
   вопросами» (двойное переключение) в «не отправится» — механизация
   самопроверки диспатчера (правило 11, слово оператора 07-16).
2. **Ход ВВЕРХ — SubagentStop, «сдача-гейт»**: v3-б (правка позже
   зелёного прогона → блок сдачи) + кандидат: пишущий воркер трогал
   пути вне owns → факт в трек (флаг, не блок — owns декларативен
   в тексте промпта, парсинг хрупок).
3. **Сдача ОПЕРАТОРУ — Stop(main), «закрытие-гейт»**: открытые
   delegated без исхода? журнал валиден (validator в STANDALONE
   режиме — закрывает no-op вне git)? Мягкий блок/предупреждение.

## Порядок строительства — по журналу утечек (промоушен D-0063,
не по симметрии)

1. **validator standalone-режим** (приоритет: 13 полевых нарушений
   №5-B + no-op трижды подтверждён; закрывает и Stop-гейт, и
   честность полевых журналов).
2. **Диспатч-гейт DoD/манифест/лейбл** (прецеденты: дыра спеки
   t-132 — collect не назван; возвраты воркеров = двойное
   переключение).
3. **v3-а/v3-б** — уже в сборке (t-150, kit v3).
4. Остальное — по мере УЛИЧЕННЫХ утечек, каждый гейт с прецедентом
   в журнале/экзамене, не «на всякий случай» (Rule #1).
