## Диагноз

Сегодняшний онбординг тулкита — «копия штаба»: политика + роли +
журнал + гейты. Для проекта с живой агентной схемой копия либо
отторгается (как у AO3 — лучший случай), либо ломает нативные
механизмы (худший). Смежная находка уже есть: F-35 (онбординг не
собирает оси самого хоста). Новое (F-51): онбординг не умеет
ОТОБРАЖАТЬ наши механизмы на существующие эквиваленты хоста.

## Принципы (ядро замысла)

Онбординг = экзамен → карта эквивалентов → слоистая установка.

1. **Экзамен-инвентаризация хоста.** Первая задача онбординга —
   evaluation, не build (расширение D-0030/D-0070 с моделей на СХЕМУ
   хоста): по каждой ФУНКЦИИ (приёмка, журнал/учёт, эскалация,
   изоляция, калибровка) — чем хост её закрывает сегодня. Мост —
   словарь функций D-0062: мы спрашиваем «кто у вас принимает
   работу», не «где ваш judge».
2. **Карта эквивалентов (adoption ledger) — первоклассный артефакт.**
   По каждому механизму кита статус: adopt (перенять) /
   native-equivalent («признанное отличие»: функцию закрывает
   нативный механизм хоста, фиксируется соответствие) / deferred
   (нет предпосылок; триггер переоткрытия назван) / rejected
   (Rule #1 хоста). Блок AO3 — стихийный прототип; продукт даёт
   шаблон. Ledger аудируем калибровкой хоста (живость соответствий).
3. **Замыкание предпосылок.** Каждый кит-механизм несёт манифест
   предпосылок (расширение R10(в) на поставку): судья требует
   калибровочную сетку, гейты — хук-путь, учёт — носитель цен.
   Установка механизма без его предпосылок запрещена конструкцией
   онбординга: либо связка целиком, либо native-equivalent/deferred
   с записью в ledger. Ровно аргумент (в)-(г) AO3, превращённый из
   их зрелости в наше свойство поставки.
4. **Measure-first, warn-first; носители хоста не мигрируются.**
   Первый слой установки — только учёт/журнал-адаптер ПОВЕРХ их
   носителей (их журнал остаётся их носителем — D-0082-логика);
   гейты приходят в warn-режиме и промоутятся в block по ИХ
   evidence утечек (D-0063). «Улучшать, не ломая» = сначала показать
   числа на их трафике, потом предлагать enforcement.
