## Находки об инструментах (в очередь)

1. **journal_validator no-op вне git**: в не-git песочнице «штатный»
   прогон молча exit 0 — ложный «валиден» (вскрыто обоими
   B-грейдерами; прямой decide() №5-B: 13 нарушений — 10
   accepted/rejected без category, повторный delegated critic без
   attempt, 2 висячие defect_found ссылки; №6-B: 3 замечания).
   Класс: enforcement, который врёт ОК вне своей среды. Фикс:
   валидатор обязан ГРОМКО сообщать «проверять нечего» (не exit 0
   молча) + песочrespawn-деплой без git — известная среда тулкита.
2. **Формат журнала не держится в полевом деплое** (13 нарушений
   №5-B): политика-текст без pre-commit гейта (нет git) не
   удерживает поля. Связка с находкой 1 — тот же корень.
3. **Пустой stdout при rc=0** (№6-C s2): не отличить «сессия молча
   сработала» от «вывод потерян» — кандидат: раннер помечает
   пустой stdout предупреждением в run_log.
4. Грейдер №6-B нарушил свой манифест (изменил state-файлы,
   восстановил контентно, не байтово; честно доложил) — песочница
   №6-B считается «восстановленной»; перепроверка её проб при
   споре — по логу проб.
