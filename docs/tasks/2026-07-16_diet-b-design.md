## Гипотеза

Диет-версия политики достигает качества полного B-Fable
(скаляр ≥0.8 табеля 6б) при ≤60% его цены, потому что качество
покупали дешёвые компоненты (вопросы вверх $1.6–2.3; witness-тесты;
детерминированные проверки), а большинство расхода — координационный
оверхед ($10–17 main + микро-веера), не производящий деливерабл.

## Состав диет-B (что остаётся / что режется — по статьям декомпозиции)

ОСТАЁТСЯ (окупилось в №5/№6):
1. **D-0077 вопросы вверх** + прокси-протез — самая окупившаяся
   статья ($1.6–2.3, выиграла спорные пробы №6-B).
2. **Witness-дисциплина**: сдача с тестами и прогоном (единственная
   клетка с тестами = лучшая по качеству); DoD-батарея правила 11
   (края ввода) как само-прогон исполнителя.
3. **Scout-разведка** (копейки, $0.35–0.69).
4. НОВОЕ (урок развилки): «что оставляешь — ревьюишь как своё» —
   аудит унаследованного кода наравне с новым.

РЕЖЕТСЯ/МЕНЯЕТСЯ (главные едоки):
1. **Батч-журналирование**: события журнала копятся и пишутся ОДНИМ
   Edit-ходом на границе сессии/задачи (сейчас — отдельный ход на
   событие; №5-B: 193 main-хода на 4 сессии). Формат строк тот же —
   меняется только каденция записи.
2. **Critic — по триггеру правила 3 и «деньги — эмпирикой»**.
   УТОЧНЕНИЕ 07-16 (вопрос оператора «когда этот критик
   сработает?»): триггер — правило 3 как есть (>100 строк / деньги /
   схема / ядро / неясный баг), и на ДЕНЕЖНОЙ задаче класса набора
   №2 он горит почти на каждый содержательный дифф — частота не
   меняется, в №5-B критик и был по этому триггеру (92 хода,
   $7.21). Экономия диеты — в ФОРМЕ и ПОРЯДКЕ, не в частоте:
   (а) денежный/численный дифф критик начинает ПРОГОНОМ контрольных
   значений (≈5 ходов эмпирики: возврат уменьшает? дубль один раз?
   копейки точны?), чтение кода — только по расхождению эмпирики
   или там, где детерминированной проверки нет (архитектура,
   безопасность); (б) малые НЕ-денежные диффы — без критика,
   само-батарея DoD. СЛЕДСТВИЕ ДЛЯ ПИНОВКИ №3: выбор задачи
   определяет, какая компонента измерима — денежная фикстура мерит
   только смену формы ревью, не-денежная — только частоту; чтобы
   измерить обе, задаче нужны И денежные, И не-денежные диффы.
3. **Микро-приёмки → балк-приёмка**: приёмка результатов воркеров
   батчем на границе этапа, не отдельным циклом на каждый дифф.
4. **Метрика-цель**: ходы main-координатора на сессию — измеряются
   и публикуются в клетке (№5-B: 48/сессию, №6-B: 22/сессию;
   диет-цель: ≤15).

НЕ РЕШЕНО (развилки пиновки набора №3, решает оператор):
- Модель координатора диет-B: Fable (данные №5/№6: Sonnet текстом
  политики дисциплину не держит — 0.70 vs 0.86) или обе для
  сравнения.
- Плечи прогона: B-full vs B-diet vs C (A — опционально, база уже
  снята в №5/№6).
- Задача: новая крупная фикстура класса «правдоподобно неверное
  дёшево сдать» (режим провала дешёвых плеч — незакрытая гипотеза
  кроссовера) ИЛИ резервный генератор сайта из markdown.
