## Конструкция плеча H (гибрид: Lead-DAG + D-машинерия на листьях)

Двухэтажная конструкция — вывод серии D№1–№3 и экзаменов №5/№6:
Lead выигрывает на графе (вопросы вверх, архитектура), D-машинерия
дешевле и не хуже на листьях. H соединяет:

1. **Координатор — FABLE** (headless-сессии плеча), политика — кит H
   (актуальное ядро CLAUDE.md D-0084 + роли scout/builder/critic с
   правилом F-49 + журнал + валидатор — состав по прецеденту
   set2-кита, с теми же исключениями: скиллы штаба и mechanism_gate
   не входят).
2. **Обязанность Lead ДО исполнения** (H-инструкция кита, поверх
   политики): intake → scope → **DAG-артефакт файлом** в песочнице
   (узлы/зависимости/статусы — D-0080) → на КАЖДЫЙ лист-узел:
   категория по allocate-таблице (лестница D-0085: recon→haiku,
   реализация→sonnet, ревью→opus) + **интент-ключи приёмки узла**
   (DoD, включая адверсариальную мини-батарею узла — R11; закрытие
   урока «судья мельче грейдера»: legacy/edge-классы кладутся в
   ключи узла заранее).
3. **Исполнение листа — БЕЗ Lead в петле**: диспатч воркера яруса по
   таблице; приёмка листа — **судья** (judge-sonnet через прокси,
   CLI-инструмент judge_accept в ките: материал клетки по механике
   judge_client с фиксами t-245 — пересчёт исключений на каждый суд,
   листинг нетронутых baseline-имён) по интент-ключам узла; reject →
   ретрай исполнителем с feedback; второй reject → эскалация НА ОДНУ
   ступень лестницы; reject после эскалации → узел failed, возврат
   Lead'у (зеркало R6 — как в D-серии). Lead лист НЕ ревьюит и не
   принимает сам (его приёмочное чтение — то, что D-серия сняла с
   пути; журнальное правило «приёмка сверху» исполняется судьёй как
   входом + финальной интеграцией).
4. **Интеграция — Lead**: сборка целого, санити против намерения
   пользователя, критик-вход на интеграцию по правилу 3 политики
   (>100 строк / схема данных — трекер это заведомо); вопросы уровня
   пользователя — ТОЛЬКО вверх (D-0077, протез ниже).
5. **Учтённые проблемы серии** (все включены): (а) галлюцинация
   судьи → reject-основания листьев проверяются форензикой на
   грейдинге (sha256/факты — метод t-244/246/247), систематика →
   находка judge-калибровки; (б) артефакт baseline-невидимости —
   закрыт кодом t-245; (в) wall-clock — независимые листья DAG
   параллелятся (лаунч t-249-класса; последовательность только по
   рёбрам зависимостей); (г) потеря evidence — деливераблы и DAG
   файлами (D-ось табеля), финал-сообщения воркеров целиком (F-49 в
   ролях кита); (д) прокси-подмены — замер моделей прокси-ходов
   cc_usage при разборе (D-0083); (е) judge-стоимость — по-вызовный
   header-учёт, traffic_kind=judge, в цене плеча.

3. РАЗВИЛКА ИНТЕРПРЕТАЦИИ (главная): «календарь» = просмотр

4. Архитектурные выборы (CLI/web/GUI, источник дат) названы.
5. Тесты либо явная демонстрация корректности; адверсариальная

  внутри плеча задачи T-A/T-B параллельны, окна по-задачные).

## Предрегистрированные ключи (числа/сравнения запинованы ДО прогона)

Сравнение — С ПЛЕЧОМ C ТОГО ЖЕ ПРОГОНА (слово оператора; та же
логика, что ключ (а) D-серии). Записанная точка №6 (B $36.46/0.86;
C $28.14/0.82) — справочная, ключом не является.

- (h-а) **Деньги**: вал H по T-A ≤ вала C по T-A того же прогона
  (судья и все ретраи H — в цене H).
- (h-б) **Качество** (не торгуется): скаляр 6б H по T-A ≥ скаляра C
  − 0.05; скаляр H < C − 0.05 = провал экзамена H при любых
  остальных. Выигрыш качества = H ≥ C + 0.05.
- (h-в) **Время**: wall-clock H по T-A ≤ wall C по T-A (метод
  cc_usage min..max, правило атрибуции — как в серии).
- (h-г) **Координационная доля** (гипотеза гибрида, измеряемая):
  доля main-ходов/токенов Fable-координатора H НИЖЕ, чем у
  Fable-сессий C (и справочно ниже №6 main ≈ 47%); отчёт числом;
  ключом не является (первый замер класса).
- T-B «Календарь» — прямое сравнение H vs C того же прогона по
  интент-ключам T-B: $, скаляр, поведение развилки (кто спросил),
  число координационных ходов; ориентир перекоординации H: цена
  клетки в коридоре D-серии t1 ($1.7–2.4) — расхождение вверх в
  разбор.
- ВЕРДИКТ: H выигрывает при (h-а)+(h-б)+(h-в); «не проигрывает» =
  паритет цены/времени при качестве ≥ C (тоже позитив — тогда
  решает структурность: журнал, DAG, персистентный evidence).
- Одиночный прогон дорогого класса — вердикты с пометкой «одиночный»
  (медианы — на повторах по кадансу большого экзамена).
