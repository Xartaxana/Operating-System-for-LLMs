## Намеренные расхождения (не дефекты синка)

1. **Язык/референсы**: toolkit — EN; RU-дрейф корня НЕ портирован
   (preflight_quota: RU-комментарии живут в КОРНЕ, toolkit-версия
   корректнее — обратное направление дрейфа, находка дизайнера).
   Наши task-id (t-NNN), даты инцидентов, форензика вычищены;
   короткие D-#### токены — по конвенции конкретного файла
   (journal_validator де-референсирован полностью).
2. **VALIDATED_DELEGABLE_CATEGORIES = frozenset()** (пусто): свежий
   деплой ничего не валидировал; наш набор сломал бы их чек-зеркало
   в день установки (вердикт B-1).
3. **regression_set_coding.jsonl**: наш набор (15 промптов = реверс
   нашего кода + наши task-id) НЕ шипается (утечка D-0070); в шаблоне
   3 генерических промпта (ISO-date parser / LRU cache / todo-CLI),
   source_task=sample-1..3.
4. **DECISIONS.md — пустой шаблон** (дом решений пользователя);
   строка D-0076 внесена пакетом C и РЕВЕРТНУТА при приёмке
   (вердикт G перевёрнут на evidence: toolkit де-референсирован по
   построению, единственная D-0076-ссылка — сама строка).
5. **config.template.yaml** vs наш config.yaml: алиасы-примеры
   (lead/critic/builder/scout/judge/analyst/mock) без наших ключей.
6. **calibration_counts: миграционные cutover'ы = эпоха**
   (1970-01-01) с комментарием «выставь свои при миграции» — наши
   даты схемы журнала не шипаются; тест переведён на
   monkeypatch-cutoff.
7. **savings_report: --routed-start стал required** — дата деплоя
   роутинга у каждого своя, нашего дефолта нет.
8. **CATEGORY_TO_TASK_TYPE** адаптирован под wording ИХ таблицы
   (строки Formatting у них нет — нейтральная метка); пост-t-095
   метки не несущие (кодового пути записи в таблицу нет).
9. **Ссылки на оси SIBLING_MAP генерализованы** (их карта — 2 другие
   оси); нумерация чеков: корневой чек 5 = их чек 4 (сверено).
10. **Принятый остаточный провенанс** (решение Lead, не чувствителен):
    ISO-таймстемпы-иллюстрации формата в preflight_quota; даты
    D-0030-оценок в guard.py; имя фикстуры t013.db; функциональные
    model-id ключи прайс-словаря usage_report (публичные id;
    удаление = регрессия прайсинга).
