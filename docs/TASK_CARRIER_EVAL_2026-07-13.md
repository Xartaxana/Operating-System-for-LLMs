## Ретроспективная модель на реальной задаче (требование гейта)

Цепочка API-окна t-075..t-080 (2026-07-12..13, 3+ сессии) — та, где
случился драйвер F-36 (шаг «в очередь Lead» потерялся между
сессиями) и рядом с которой жил t-029 (дубль task_id на границе
рестарта). Как выглядел бы DAG-артефакт:

```
узел t-074 recon API-окна        [done]  ярус: scout
узел t-075 кэш-колонки логгера   [done]  ярус: builder  блокирует: t-077,t-078,гейт-C3
узел t-076 recon стриминга       [done]  ярус: scout    блокирует: t-077
узел t-077 стриминг-тест         [done]  ярус: builder
узел t-078 metrics-кэш           [done]  ярус: builder  (attempt 2 после rejected)
узел t-079 D-0075 самотег        [done]  ярус: builder
узел t-080 фикс F-38             [done]  ярус: critic   ref: t-078
шаг  «порт кэш-колонок в toolkit» [queued] ярус: Lead   ЖДЁТ: закрытие Phase 2  <- F-36-класс
```

Проверка против драйверов: (а) F-36-шаг в этой форме имеет явный
узел со статусом queued и триггером — не растворяется в прозе
секции; (б) t-029-класс (какие id заняты) виден списком узлов рядом
с max(t-NNN) журнала. Обе потери, давшие P2, этим артефактом
покрываются. Стоимость модели: ~8 строк на задачу из 7 узлов —
К5-замер для пилота.

## Вердикт

1. **Нативные task-тулы — НЕ носитель** (отвергнуты как первичный
   носитель): проваливают К2 (вне git — вне памяти проекта, D-0001)
   и межмашинность К7 по построению. ДОПУСТИМЫ как внутрисессионная
   рабочая доска (их DAG-механика лучшая), но source of truth они
   быть не могут; всё пережившее сессию обязано материализоваться в
   git. REOPEN-ТРИГГЕР: project-level tasksDirectory (открытый
   feature request anthropics/claude-code#20425) — при появлении
   пересмотреть, К2 снимется.
2. **Markdown-артефакт в репо — РЕКОМЕНДОВАННЫЙ носитель**:
   единственный кандидат, проходящий К1/К2/К4/К6/К7 по построению;
   К3/К8 выразимы; открытый риск — К5 (ручное поддержание, третье
   место записи) — меряется пилотом, порог: упорство дисциплины
   проверяет калибровка (дрейф узлы↔журнал).
3. **Статус-кво недостаточен подтверждённо**: он уже дал P2-драйверы
   (F-36, t-029) и грузит boot-файл (К6 — пять диет за неделю).
