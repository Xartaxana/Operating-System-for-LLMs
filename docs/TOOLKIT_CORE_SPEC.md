## Онбординг (скилл onboarding; «минимум внимания» с первого шага)

1. **Один вопрос:** «Working on a Claude Code subscription, or on a
   set of API keys from different providers?» → контур выбирается
   сам: subscription = политика+агенты+журнал; api-keys = то же +
   gateway (алиасы из привязки). Оба сразу — тоже валидный ответ.
2. **Привязка моделей — один файл** `delegation.config.yaml`:
   `scout:`, `builder:`, `critic:`, `lead:` (+ `judge:`, `analyst:`
   для API-контура) → онбординг раскладывает в frontmatter агентов
   и/или gateway config. Словарь правил не меняется (D-0062).
3. **Экзамены (В6: обязательны, провал переопределяем словом):**
   - scout: golden set ГЕНЕРИРУЕТСЯ под репо пользователя скиллом
     scout-exam-gen (процедура D-0057: 7 вопросов, пиновка ключей
     СВОИМИ командами Lead-модели пользователя, negative и
     judgment-refusal обязательны), прогон немаркированным диспатчем;
   - lead-кандидат: LEAD_RANKING_EXAM виньетки (портируемы);
   - judge (API-контур): judge_calibration.json (портируем), порог
     из JUDGE_CALIBRATION_PROTOCOL;
   - не-Claude воркеры: pi_run_guard на первом прогоне.
   Провал → «Model X failed the [scout] exam (2/7; mandatory trap
   failed). A stronger model is recommended.» + вопрос: заменить или
   оставить своим словом; «оставить» пишется в журнал деплоя событием
   (operator override, model, score) — калибровка это видит.
4. Инициализация: hooksPath, журнал, первый Boot Report.

## Контракт уведомлений (В5 интейка → продуктовое правило)

Фоном без пинга: SessionStart-печать, Boot Report, журнал, фоновые
диспатчи. Пользователя дёргают ТОЛЬКО: (1) две неудачи приёмки на
верхнем ярусе / эскалация выше некуда; (2) BREACH бюджета/квоты;
(3) провал экзамена (онбординг или смена модели); (4) недельный
дайджест калибровки одним сообщением. Всё.

## Перевод EN — самый большой строительный кусок

Политика/роли/скиллы/протоколы сейчас RU. Перевод обязан сохранить
НОРМАТИВНУЮ семантику (каждое правило — с номерами решений класса
поведения, но БЕЗ ссылок на наши D-номера истории: в шаблоне решения
пользователя начнутся со своего D-0001; наши D-ссылки заменяются
описательными якорями правил). Порядок: политика → роли → скиллы →
протоколы; каждый файл builder-переводом + critic-ревью на
семантическую эквивалентность против RU-оригинала.
