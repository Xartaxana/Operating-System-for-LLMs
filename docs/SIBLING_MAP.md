# Sibling Map — карта осей симметрии (D-0043)

Инструмент правила «чини класс, а не экземпляр»: закрывая дефект, обход
аналогичных мест делается ПО ЭТОЙ КАРТЕ, а не сканом репозитория.
Карта намеренно мала и грузится только в момент закрытия дефекта (не
при boot — D-0038). Порядок работы:

1. Найди ось (оси), на которых лежит дефект — см. список ниже.
2. Проверь/почини парные места оси точечно; что не чинится сейчас —
   явно в очередь (CURRENT_CONTEXT.md) или журнал.
3. Класс шире карты или ось не находится → диспатчь scout (haiku) на
   точечный поиск с конкретным вопросом. Lead/critic НЕ перечитывают
   репозиторий руками (правило 1).
4. Появилась новая симметрия (новый деплой, новый парный документ,
   новая пара код/учёт) → добавь ось сюда В ТОМ ЖЕ коммите — это
   часть шага 3 D-0043.

critic проверяет классовую полноту фикса ПО КАРТЕ: «покрыты ли оси,
на которых лежит дефект» — ограниченный вопрос по диффу и карте, не
по всему коду.

## Ось 2 — Контуры измерения (Rule #1, D-0032: учёт честен на обоих)

| API-контур (gateway) | Субскрипционный контур (Claude Code) |

Классические классы на этой оси: новая модель/цена (добавить в ОБА
учёта), новый вид трафика/атрибуции, правило «никогда молчаливый $0».

## Ось 3 — Роли/ярусы (обязанность одной роли — проверь остальные)

scout / builder / critic (х2 деплоя) + Lead-правила в CLAUDE.md +
10 QA-агентов AO3 (их промпты — цикл изменений конвейера; правки туда
ставятся в очередь, если не по пути). Пример класса: «докладывай
замеченные аналоги» (D-0043), «не запускай субагентов» (D-0037).

## Ось 4 — Парные документы (инвариант: не расходиться)

- DECISIONS.md <-> CLAUDE.md обоих деплоев (решение принято — политика
  отражает; F-9/D-0042 были рассинхроном именно здесь).
- ROADMAP.md — единственный владелец фаз/гейтов; ARCHITECTURE.md и
  WHITE_PAPER.md ссылаются, не дублируют.
- BOOT.md <-> MEMORY_ARCHITECTURE.md (слои и список загрузки).
- docs/FINDINGS.md (наши наблюдения) <-> docs/RELATED_WORK.md (внешние
  приоры): факт заходит в один из двух, по происхождению.
- CURRENT_CONTEXT.md <-> docs/task_reports/ (D-0038: закрыто — в архив,
  указатель остаётся).
