# Ретроспективные паттерны — реестр (2026-07-13)

Единственный владелец бэклога ретроспективных паттернов калибровки.
Родился из обзора harness engineering / Case (docs/RELATED_WORK.md,
секция 2026-07-13) и аудита протокола по вопросу оператора «чиним ли
мы машину так, чтобы минусы не повторялись, и ищем ли, что можно
сделать лучше». Порядок — по важности/полезности (слово оператора
2026-07-13). Внедрённый паттерн живёт ЧЕКОМ в
PROCESS/WEEKLY_CALIBRATION_PROTOCOL.md — здесь остаётся строкой со
ссылкой (пара оси 4 SIBLING_MAP: перенос из бэклога в протокол
двигает строку в «Внедрено» тем же коммитом). Триггеры бэклога
сверяет чек 0 каждого прогона: триггер выстрелил, а паттерн спит =
находка.

## Бэклог (по важности; до именованного триггера не строим — Rule #1)

1. **Знание осталось в чате** (детектор D-0014): скан транскриптов
   окна на решения/уроки, не вынесенные в git. ТРИГГЕР: первый
   прецедент невынесенного знания (находка) либо провал ZCRT-класса.
2. **Overqualified dispatch**: задачи дорогого яруса, принятые с
   первой попытки с тривиальным следом (короткий сайдчейн, малый
   дифф) — кандидаты понижения категории; зеркало чека 19 (таблица
   сейчас умеет двигаться вниз только через defect_found). ТРИГГЕР:
   третья калибровка (нужны >=2 окна тренда); форма — расширение
   чека 19.
3. **Doom-loop / повтор тул-вызова без изменений** (Case): скан
   транскриптов на петли (тот же вызов >=3 раз подряд без изменения
   входа). ТРИГГЕР: первый замеченный инцидент петли.
4. **Задачи-сироты**: delegated без accepted/rejected, пережившие
   границу сессии (чек 6 смотрит только завершённые). ТРИГГЕР:
   первый висяк.
5. **Правка-откат (churn)**: файл правится и переписывается/
   откатывается в одном окне — сигнал слабой спеки или недочитанного
   контекста. ТРИГГЕР: систематический failure_class=spec (сигнал
   чека 13г).
6. **Permission-денаи ретроспективно**: повторные отказы разрешений
   на одной форме команды → команда в гигиену/allowlist. ТРИГГЕР:
   повторный денай одной формы за окно, замеченный сессией.
7. **Потерянный параллелизм** (Case): последовательные независимые
   диспатчи там, где легален веер (правило 4). ТРИГГЕР: вместе с
   №3 — один и тот же скан транскриптов.
8. **Cache-аномалии сессий**: сессии с аномально низкой долей
   cache-read (разрывы >5 мин TTL, рестарты) — дорогая организация
   работы, невидимая в суммах чека 18. ТРИГГЕР: рост $/день (чек 18)
   без роста объёма работы.
9. **Рост retroactive-событий журнала**: журнал ведётся задним
   числом, ts-честность (F-29) под давлением. ТРИГГЕР: >=2 ретро-пар
   за окно.
10. **Время до первого диспатча**: Lead долго работает сам до
    первого делегирования — мягкий прокси самоизъятия. ТРИГГЕР:
    подозрение самоизъятия при пустом чеке 1.
11. **Мёртвые estimated-строки таблицы**: категории без единого
    evidence-события N окон подряд — чистить или признать
    выдуманными. ТРИГГЕР: третья калибровка.
12. **A/B-eval механизма качества** (Case: скилл давал 77% против
    97% без него — узнали только замером): механизм с претензией на
    улучшение КАЧЕСТВА (не дисциплины) получает малый замер «с ним
    vs без». ТРИГГЕР: следующий такой механизм; принцип — в
    eval-план stage 2.
